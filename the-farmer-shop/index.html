Current Site Overview

WooCommerce store for an Australian farming/agricultural supplies and companion-animal products retailer. Catalog spans 11 categories (Animal Identification, Bird Products, Cattle, Companion Animals, Equine, Sheep Breeding, Farm & Hardware, Feeding & Watering, Veterinary Supplies, Pest Control, Poultry Equipment). Site includes a mega-menu navigation, homepage hero/collection banners, an "About" section, customer testimonials, a "What's New" section, and an FAQ accordion.

Branding, layout, and user flow are already solid — the main gaps are in product-page/collection UX polish (filters, sticky add-to-cart, minicart quantity control, mobile header/layout fixes) rather than the overall structure. See Section 4 for the full review; these fixes are already folded into Option A's scope below.

Two Implementation Directions

Two directions are quoted below so the client can choose.

Option A

Replicate

Clone the current site's layout and structure (navigation, homepage sections, category/product page layout, footer) onto Shopify using an existing/free Shopify theme's default layout as the build foundation; migrate products, categories, and site content from WooCommerce to Shopify. This scope already includes implementing the team's recommended UX/design fixes listed in Section 4 (icon/button polish, product card contrast, mobile header redesign, mobile overflow fix, collection filters, sticky add-to-cart, bundle/frequently-bought-together, minicart quantity control, header search) — this is our own recommended approach to a proper replicate, not a bare 1:1 clone of the current site's rough edges.

Option B

New Design (based on the theme's default layout)

Built on the recommended theme's own default layout — not a custom layout. Instead of forcing the current custom design onto Shopify, this direction starts from a well-matched premium theme's out-of-the-box layout and customizes it to the client's branding (logo, colors, fonts, homepage sections), rather than rebuilding the current page structure. This route typically lands cheaper than Option A, because several of the same fixes (quick view, sticky add-to-cart, mega menu, product filters, cart-drawer quantity control) come built into the theme's default layout rather than needing custom development.

Recommended theme: Impulse (Archetype Themes, premium license). Fits this catalog well because it ships out-of-the-box with: mega menu (matches the current 11-category nav), product filters, Quick View, sticky Add to Cart, image galleries with hover/zoom, complementary-products blocks (covers the "Frequently Bought Together" ask), and a cart drawer with quantity controls. Since theme cost isn't a constraint for the client, this is the strongest fit rather than a free/basic theme that would need custom work to add the same features. (Alternative if a second opinion is wanted: Motion or Warehouse — similar feature set, worth a quick comparison before committing.)

Apps/functions to flag

  • Search & Discovery (Shopify's own, free) — covers both the Ajax/predictive search request and collection filters natively; recommended over building either custom.
  • Frequently Bought Together / Bundle app — a paid app (typically a low monthly fee) would still be needed even with Impulse's complementary-products block, if the client wants the "Bundle & Save" discount mechanic specifically rather than just a suggestion block.
  • Quick View, hover-image swap, sticky Add to Cart, and cart-drawer quantity stepper are native to Impulse's default layout (Option B, no extra build time) and are already custom-built into Option A's 45h (see Section 2) — either option delivers them, just via a different route.

Design & UX Fixes Included in Option A

These are the specific fixes referenced in Section 2 above — already part of Option A's 45h scope, and mostly covered natively by Option B's recommended theme (flagged above where they aren't).

Strengths (already working well)

  • Clear, consistent branding from header to footer.
  • Clean homepage layout — logical sections, balanced whitespace, doesn't feel cluttered.
  • Natural user flow — shoppers can find what they want without much thinking.
  • Product card baseline is solid: sharp product photos, price stands out, Add to Cart button is easy to see, information is concise.
  • Several CTAs already use a consistent style (orange background, white text, pill shape) — e.g. "Explore More," "Discover More," "View All Products." Good reference for extending this consistency everywhere.

1. Product card — quick add-to-cart icon contrast

The small circular "add to cart" icon on product cards currently shows a dark cart glyph on an orange background — low contrast. Recommended: switch the icon to white for better visibility.

2. Product card — additional polish (non-structural)

Without changing the existing card structure: ⭐ show star rating directly on the card · 👁 Quick View for a fast preview · 🖼 hover image swap (second product photo on mouse hover) · ✨ a light hover animation (subtle shadow or slight lift).

3. Product title sizing

Reduce font size by ~1–2px and slightly lighten the font-weight; cap display at 2 lines with ellipsis if too long. Keeps cards more compact, evens out spacing between cards, and lets price/Add-to-Cart stand out more.

4. Button consistency across sections

Standardize all buttons to the same size and style — solid orange background, white text — sitewide (several sections already do this well; extend it everywhere).

5. Product card background contrast

Slightly darken/warm the card background tone so cards read as distinct blocks against the page's white background instead of blending in.

6. Hover effect on category tiles

The "Favourite Categories" homepage tiles (Cattle Essentials, Companion Animals, Equine, Sheep Breeding, etc.) would benefit from a hover effect for better interactivity feedback.

7. Mobile header redesign

The current mobile header stacks logo, phone number, search icon, cart icon, and a "Contact Us" button in a white block, followed by a separate bright-yellow bar for the hamburger menu — reads as two disconnected sections. Recommended: consolidate into a single, cleaner mobile header.

8. Mobile layout bug — homepage overflow

On mobile, the "Poultry Equipment" (and similar) homepage promo sections overflow horizontally — heading and "View All Products" button get cut off at the viewport edge instead of wrapping/resizing properly. This is a genuine responsive bug to fix regardless of which rebuild direction is chosen.

9. Collection page filters

Collection/category pages currently have no filtering (price, size, etc.). Recommended: add filters either in a sidebar or above the product list. On mobile, collapse this into a filter icon that opens a popup for shoppers to pick their own filters.

10. Product Detail — Sticky Add to Cart

When a shopper scrolls down to read the description, specs, or reviews, the Add to Cart button disappears. Recommended: add a Sticky Add to Cart bar at the bottom of the screen (desktop and mobile).

11. Bundle Products / Frequently Bought Together ⭐ (recommended)

Pet-category shoppers commonly buy multiple related items in one order (e.g. food + toys). Adding "Frequently Bought Together" or "Bundle & Save" would let shoppers add grouped items with an attractive combined offer.

12. Minicart — quantity control

The current minicart design doesn't let shoppers adjust quantity directly — they have to go into the full cart page to change it, adding an extra step. Recommended: add inline quantity +/- controls in the minicart itself. A reference minicart pattern from another site was reviewed for comparison, showing what a fuller-featured drawer can include: quantity stepper, a free-shipping progress bar, a "frequently bought with" cross-sell block, and a secure-checkout/Afterpay trust strip.

13. Header search

Convert the current small search icon into a proper search form positioned centrally in the header, and add Ajax/predictive search (results-as-you-type) — a standard expectation on modern storefronts.
