Online Store Build — Layout & Functions (LitOS)

2.1 Design Phase

A design mockup (desktop view) is created before development begins, covering Homepage, Collection (Category Page) and Product Detail Page.

Up to 3 feedback rounds included for layout, spacing, section, or styling adjustments before design finalization. Development begins only after the designs are approved.

2.2 Development Phase

A. Pages & layout

Homepage, collection pages (categories), product pages, contact page, policy pages, navigation structure, footer. All pages follow the approved homepage design style and brand identity.

B. Product & category structure

Products and categories from both generalseafoodinc.com and generalseafood.shopsettings.com are consolidated into one Shopify store under a single domain. Collection and product pages use the theme's default layout, adjusted to match branding and improve usability.

C. Product page features

Product images, description, add-to-cart, variants (if applicable), price display, quantity selector.

D. Contact page

Store contact info (email, phone, address), contact form, embedded Google Map, social platform links. (If the map/store info currently duplicated on every product page applies store-wide, it moves to the Contact page instead — cleaner and easier to maintain.)

E. Design improvements

Previously discussed with client:

  • Sticky header with Facebook and Instagram links embedded
  • Visible cart icon in the header
  • Search function in the header (desktop and mobile)
  • Improved footer structure
  • Newsletter signup section
  • Homepage banners / featured sections
  • Collection page filters
  • Quick view functionality for products

F. Responsive design

Optimized and consistent across desktop, tablet, and mobile.

Shopify POS Setup (LitOS)

Based on Shopify's native POS workflow and Smart Grid structure:

A. Shopify POS configuration

Configure the POS sales channel, ensure migrated products are available in POS, verify Admin↔POS sync (product info, images, pricing, inventory, variants).

B. Smart Grid setup

Shortcuts for key collections and frequent actions, tiles organized around the client's retail workflow, POS navigation configured using Shopify's native Smart Grid.

C. Location setup

Configure retail location(s), assign inventory, verify inventory visibility in POS.

D. Initial testing and guidance.

Note on scope: Shopify POS runs on Shopify's native POS framework — it will not replicate the current Lightspeed Retail interface exactly. Product access in POS will be via Smart Grid shortcuts, collections, search, or barcode scanning rather than Lightspeed's current navigation. Worth setting this expectation with the client up front.
