Methodology & limitations

  • What was checked live: homepage, the Caramel Toffee Pu-erh PDP, and the empty cart page, fetched directly from 1t7aeh-fx.myshopify.com and inspected at the HTML/source level (link targets, meta tags, heading structure, section markup) — not screenshots, so this catches things a visual pass can miss and vice versa.
  • What couldn't be checked this way: anything requiring Shopify Admin access (inventory, taxes, discount codes, Admin-configured navigation menus) or an interactive browser session (wishlist add/remove state, sticky-scroll behavior, checkout payment flows, email triggers, GA4/Meta Pixel firing, cookie consent, contrast/keyboard testing). These are marked Needs live/admin check and estimated from the theme patterns visible in the source plus standard Shopify practice — confirm once you have store access.
  • ET basis: grounded in this theme's actual structure (a Dawn-derived theme, confirmed from class naming and section/schema conventions) and the patterns in this project's shopify-liquid-themes, liquid-theme-standards, and liquid-theme-a11y skills — not a generic Shopify estimate.
  • Next step for a tighter estimate: connect the Shopify Admin MCP integration (currently unauthenticated in this session) so inventory, tax, discount, and navigation-menu state can be checked directly instead of estimated from indirect evidence.
