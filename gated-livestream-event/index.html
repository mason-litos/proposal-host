Shopify Plus Development Proposal

Gated Livestream Event Product

Feature: restrict event product purchase to livestream QR-verified customers only
Platform: Shopify Plus Date: July 22, 2026 Rate: $25/hr

Solution Overview

The feature is technically feasible on Shopify Plus. It is not a native Shopify capability — it requires a custom app. Two architecture options are proposed below, differing in how strictly the purchase is enforced.

Option A (recommended) enforces verification at checkout with a Shopify Function — the restriction cannot be bypassed by direct API calls or express checkout (Shop Pay, Apple Pay, Google Pay). Requires Shopify Plus.

Option B (simplified) only shows/hides the buy button on the storefront based on verification state. Faster and cheaper to build, does not require Shopify Plus — but a technically capable customer could bypass verification entirely and purchase without ever scanning the QR code, since there is no server-side enforcement at checkout.

Option A — Full Solution Recommended

Checkout-level enforcement via Shopify Function. Cannot be bypassed by direct storefront/cart API calls. Requires Shopify Plus (custom apps using Shopify Function APIs are Plus-only).

Option B — Simplified Solution Faster / Lower cost

Front-end gating only (show/hide buy button). No Shopify Function, no checkout-level enforcement. Does not require Shopify Plus.

Risk to accept before choosing this option: a customer with basic technical knowledge (browser DevTools, or calling the storefront cart API directly) can purchase the event product at the special price without scanning the QR code or entering a verification code. This is an inherent limitation of front-end-only gating, not a bug that can be fully patched while keeping this simpler architecture.

Platform & Infrastructure Requirements

Shopify Plus planRequired for Option A only — custom apps using Shopify Function APIs (checkout enforcement) are Plus-exclusive. Option B can run on any standard Shopify plan.
Custom app hostingRequired for both options — Shopify does not host the app backend. A separate server (e.g. Fly.io, Render, Vercel) is needed, with ongoing hosting cost not included in the one-time development estimate above.
Verification code lengthA 4-character code has a very small keyspace and can be brute-forced in seconds without rate limiting. Both estimates above include rate limiting; a minimum of 6 alphanumeric characters is recommended if UX allows.
Instant code deactivationManual deactivation by the admin takes effect within 15–30 minutes, not instantly, on a standard Shopify Plus plan. True real-time revocation requires Shopify Plus Enterprise (network access inside Shopify Functions) — an additional cost to consider if instant revocation is a hard requirement.

Note: the estimates above are preliminary, intended for scoping and budget discussion. Final effort should be confirmed after a detailed requirements review with the client.
